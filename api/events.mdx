The Events API provides access to audit logs and activity events in your NetBird network, enabling compliance, monitoring, and security analysis.

## Audit Events

Audit events track administrative actions and system changes.

### List All Audit Events

```bash
GET /api/events/audit
```

<CodeGroup>
```bash Request
curl -X GET https://api.netbird.io/api/events/audit \
  -H "Authorization: Token nbp_YOUR_TOKEN"
```

```json Response
[
  {
    "id": "10",
    "timestamp": "2023-05-05T10:04:37.473542Z",
    "activity": "Route created",
    "activity_code": "route.add",
    "initiator_id": "google-oauth2|123456789012345678901",
    "initiator_name": "John Doe",
    "initiator_email": "demo@netbird.io",
    "target_id": "chad9d86lnnc59g18ou0",
    "meta": {
      "name": "my route",
      "network_range": "10.64.0.0/24",

    }
  }
```

</CodeGroup>

<ResponseField name="id" type="string">
  Unique event identifier
</ResponseField>

<ResponseField name="timestamp" type="string">
  When the event occurred (ISO 8601 format)
</ResponseField>

</ResponseField>

  Machine-readable activity code
</ResponseField>

</ResponseField>

<ResponseField name="initiator_email" type="string">
  Email of the user who triggered the event
</ResponseField>

<ResponseField name="target_id" type="string">
  ID of the resource affected by the event
</ResponseField>

<ResponseField name="meta" type="object">
  Additional event metadata
</ResponseField>

## Network Traffic Events

<Note>
Network traffic events are a Cloud-only experimental feature that tracks network flows between peers.
</Note>

### List Network Traffic Events

```bash
GET /api/events/network-traffic
```

<ParamField query="page" type="integer" default="1">
  Page number
</ParamField>

<ParamField query="page_size" type="integer" default="1000">
  Items per page (max 50000)
</ParamField>

<ParamField query="user_id" type="string">
  Filter by user ID
</ParamField>

<ParamField query="reporter_id" type="string">
  Filter by reporter (peer) ID
</ParamField>

<ParamField query="protocol" type="integer">
  Filter by protocol (1=ICMP, 6=TCP, 17=UDP)
</ParamField>

<ParamField query="type" type="string">
  Filter by event type: `TYPE_START`, `TYPE_END`, `TYPE_DROP`
</ParamField>

<ParamField query="connection_type" type="string">
  Filter by connection: `P2P` or `ROUTED`
</ParamField>

<ParamField query="direction" type="string">
  Filter by direction: `INGRESS`, `EGRESS`
</ParamField>

<ParamField query="search" type="string">
  Search user email, source/destination names and addresses
</ParamField>

<ParamField query="start_date" type="string">
  Start date filter (ISO 8601)
</ParamField>

<ParamField query="end_date" type="string">
  End date filter (ISO 8601)
</ParamField>

<CodeGroup>

```bash Request
curl -X GET "https://api.netbird.io/api/events/network-traffic?page=1&page_size=100" \
```

## Reverse Proxy Access Logs

View access logs for reverse proxy services (NetBird Cloud feature).

### List Proxy Access Logs

```bash
GET /api/events/proxy
```

<ParamField query="page" type="integer" default="1">
  Page number
</ParamField>

<ParamField query="page_size" type="integer" default="50">
  Items per page (max 100)
</ParamField>

<ParamField query="sort_by" type="string" default="timestamp">
  Field to sort by: `timestamp`, `url`, `host`, `path`, `method`, `status_code`, `duration`, `source_ip`, `user_id`, `auth_method`, `reason`
</ParamField>

<ParamField query="sort_order" type="string" default="desc">
  Sort order: `asc` or `desc`
</ParamField>

<ParamField query="search" type="string">
  General search across fields
</ParamField>

<ParamField query="host" type="string">
  Filter by host header
</ParamField>

<ParamField query="method" type="string">
  Filter by HTTP method
</ParamField>

<ParamField query="status_code" type="integer">
  Filter by HTTP status code
</ParamField>

<ParamField query="start_date" type="string">
  Start date filter (RFC3339 format)
</ParamField>

<ParamField query="end_date" type="string">
  End date filter (RFC3339 format)
</ParamField>

```bash Example
  -H "Authorization: Token nbp_YOUR_TOKEN"
```

## Event Monitoring Strategies

### Security Monitoring

Monitor security-relevant events:

```bash
# Watch for user account changes
activity_code: user.role.update
activity_code: user.block
activity_code: user.create

# Watch for policy changes
activity_code: policy.delete
activity_code: rule.update

# Watch for setup key usage
activity_code: setupkey.overuse
activity_code: peer.setupkey.add
```

### Compliance Auditing

Track administrative actions:

```bash
# Configuration changes
activity_code: account.setting.*
activity_code: network.*
activity_code: route.*

# Access control changes
activity_code: policy.*
activity_code: group.*
```

### Operational Monitoring

Monitor network health:

```bash
# Peer connectivity
activity_code: peer.user.add
activity_code: user.peer.delete
activity_code: peer.login.expire

# DNS changes
activity_code: dns.zone.*
activity_code: nameserver.group.*
```

## Integration Examples

### SIEM Integration

Forward events to your SIEM:

```python
import requests
import time

def fetch_events():
    response = requests.get(
        "https://api.netbird.io/api/events/audit",
        headers={"Authorization": f"Token {token}"}
    )
    return response.json()

# Poll for new events every minute
while True:
    events = fetch_events()
    for event in events:
        send_to_siem(event)
    time.sleep(60)
```

### Webhook Notifications

Send critical events to Slack/Discord:

```python
def check_critical_events(events):
    critical_codes = [
        "user.role.update",
        "policy.delete",
        "account.delete"
    ]
    
    for event in events:
        if event["activity_code"] in critical_codes:
            notify_team(event)
```
